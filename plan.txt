The system currently redirects newly registered owners to 
dashboard_info_edit.html
 via owner.profile. That template is for replacing the customer's personal info and it hardcodes the form action to post to customer.edit_profile (thus triggering "Access denied. Customer only"). Also, it lacks the necessary restaurant fields (name, contact, address, description, logo).

Create a new template 
owner_profile.html
 that extends 
base_owner.html
.

contact
, address, description, and 
logo

app/routes/owner.py
 
profile

 instead of

2. Sign Up Form Validation Display
The red "Invalid email address" message (and other validation errors) look squished and overlap the form fields because of a hardcoded inline style: margin-top: -15px.

Action:

Modify 
app/templates/sign_up.html
 to remove margin-top: -15px from the error span elements, and apply standard Bootstrap text margins (e.g., mt-1 mb-2).
3. Checkout Validation Constraints
The system currently allows customers to check out even if their address and phone number are empty.

Action:

Modify 
app/routes/customer.py
 
checkout()

app/templates/owner_media.html

dashboard.html
 / restaurant.html) to display these promotional images if present (e.g., in a carousel or gallery on the restaurant details page).
